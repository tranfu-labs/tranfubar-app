// 产品

个人端负责采集，团队端负责看全局。

第一版只接 Codex 和 Claude。个人端不上传 API Key、Cookie 或对话内容，只上传套餐、额度窗口、Token 汇总和重置时间。

个人版 TranFuBar

安装在团队成员自己的 Mac 上，读取本地 Codex / Claude 登录态和日志，显示个人剩余额度，并把汇总数据上报团队服务。

需要填写
团队服务地址、团队 ID、成员名称、节点名称、团队 Token
默认频率
跟随 CodexBar 刷新频率，可在设置里调整
不采集
真实 API Key、Cookie、对话内容、源码内容
团队版 TranFu Usage Team

部署在团队服务端，接收每个 TranFuBar 节点上报的数据，提供成员用量、供应商占比、额度告警和 Agent 化程度评估。

需要准备
域名、服务器、HTTPS、SQLite、部署账号
建议补齐
团队 token、管理员入口、备份策略、运行日志
官网部署
页面风格已按 tranfu.com 的产品/资源布局靠拢

团队版上线前置清单

v1
服务器一台可公网访问的 Linux 服务器，先跑 Node 服务即可。
域名tranfubar-app.tranfu.com 解析到服务器，并配置 HTTPS。
数据存储正式环境使用 SQLite，便于部署、备份和排查。
团队标识统一 teamId：tranfu-ai，个人端都填同一个。
成员信息每个人填写成员名和节点名，便于团队面板识别。
安全策略上报接口使用团队 Token，避免公网接口被随意写入。
